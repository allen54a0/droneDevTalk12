# EP02 — 會飛 ≠ 能出任務｜九成新創團隊死在這個坑

(點選觀看影片)
[![影片標題](https://img.youtube.com/vi/WDs2etCLZ1M/maxresdefault.jpg)](https://www.youtube.com/watch?v=WDs2etCLZ1M)

大家好，我是阿仁。「現在開源飛控那麼成熟，YouTube 上高中生都能組飛機，買一台大一點的飛機裝上飛控、GPS、通訊、相機，不就能出任務了嗎？」這是我被問過無數次、也是最多人誤判的問題。這一集，我們把「飛得起來」跟「能完成任務」這兩件事徹底拆開——為什麼這是無人機新創團隊最容易「陣亡」的一個坑。

## 本集重點

### 1. 會飛只是解決飛行問題，不是解決任務問題
飛得穩、繞兩圈、待一段時間——現在做一台會飛的玩具，門檻真的很低。但任務問的是完全不同的問題：在 100 公里外要看多遠、要看清楚車牌還是人臉、要不要即時回傳、資料給誰用。這些幾乎沒一個是飛控電腦能單獨回答的。會騎機車，不代表你能在颱風天送外送還保證湯不灑出來。

### 2. 任務型無人機，必須在同一次飛行中同時驗證所有能力
工廠裡可以分項測試打勾，但 POC 現場是「同一次飛行，同時驗證飛行、滯空、通訊、酬載、人機協作、資料交付」。其中一個環節掉鏈子，比如飛回來了但照片沒存到，整趟任務就是 0 分。系統工程裡沒有 90 分這種事。

### 3. 真正花時間的，從來不是把飛機做出來
設備選型 1–3 個月、訂貨等待 1–6 個月（非紅供應鏈半年是常態）、學習新設備、次系統與全系統整合 1–6 個月。一個成熟的系統，半年起跳、一年常態，絕對不是三天速成班可以搞定的。請把「等待與整合」的時間算進去。
