# EP09 — 為什麼 80% 的失敗不是壞在飛機？談 POC 現場的生存法則

(點選觀看影片)
[![影片標題](https://img.youtube.com/vi/8O3F2yPHPrU/maxresdefault.jpg)](https://www.youtube.com/watch?v=8O3F2yPHPrU)

大家好，我是阿仁。每家廠商在 POC 那天，紙上規格看起來都差不多——能飛 100 公里、能掛高階光電、能抗六級風。但到了海邊，為什麼有的飛機飛得像信天翁，有的卻像喝醉酒？老實說，**80% 的失敗，都不是敗在飛機本身，而是對「驗證」這兩個字的誤解**。這一集，我們聊整合驗證——無人機開發中最驚心動魄、也最燒錢的環節。

## 本集重點

### 1. 規格 ≠ 任務能力
工廠裡的數據是靜態的（恆溫 25 度、無干擾、無側風），標案現場是動態的（海面反射、地形遮蔽、七級陣風讓馬達電流瞬間飆三倍）。「整合驗證」不是考你的飛機「能不能飛」，是考你的系統「能不能在極限邊緣活下來」。

### 2. 團隊 > 機器
法拉利賽車交給剛拿駕照的工讀生開，第一個彎就出去了。無人機任務團隊至少要有三種角色：**任務指揮官**（看全局、做風險決策）、**IP 內部飛行員**（監控自動飛行、感覺到飛機在掙扎）、**酬載操作手**（看畫面、調天線、回報資訊）。沒有經過磨合的團隊，給他再好的飛機也是浪費。

### 3. 驗證要有層次，不要越級打怪
金字塔型驗證：**地面整合測試**（震動、遮擋、電壓掉壓）→ **封閉場域飛行**（測功能）→ **模擬任務驗證**（去風大的海邊、有山丘的地方測性能邊界）。記住：驗證的目的不是證明飛機「能飛」，而是找出它在什麼情況下「會出事」——在自己家裡把飛機操壞，總比在驗測官面前摔來得好。
